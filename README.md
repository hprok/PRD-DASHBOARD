# 🚀 PRD Interactive Dashboard (Prototype)
**Eliminating Documentation Friction with Interconnected Systems.**

### 📌 Overview
Static, 100+ page PRDs often lead to information silos and development delays. This project is a **self-initiated solution** designed to transform fragmented documentation into a navigable, "Wiki-style" interactive experience.

* **Objective:** To optimize how stakeholders (Developers, QAs, Designers) explore complex Use Cases, Business Rules, and UI Workflows.
* **Impact:** Reduces cross-referencing time by **~70%** through synchronized, side-by-side component previews.

### 🤖 AI-Augmented Development
This project showcases a modern **AI-Human collaboration workflow**:
* **Role:** I acted as the **Product Architect**, defining the system logic, data mapping, and user flows.
* **Implementation:** Leveraged **Google AI Studio** for rapid UI prototyping and boilerplate generation.
* **Strategy:** Utilized advanced **Prompt Design** and iterative content structuring to ensure logical consistency across the entire documentation ecosystem.

### ✨ Key Features
* **Global Hyperlinking:** Click any system entity (blue links) to instantly trigger a side-by-side preview.
* **Contextual Navigation:** Seamlessly jump between **Use Cases ↔ UI Screens ↔ API Specs** without losing context.
* **Integrated Workflow Visualization:** Direct mapping of business rules to functional screens.

### 🛠️ Tech Stack
* **Frontend:** React, Tailwind CSS
* **Intelligence:** Google AI Studio (Gemini Pro)
* **Architecture:** Component-based UI for modular documentation.

---

### 🚀 Setup & Installation
1.  **Clone the Repo**
2.  **Install dependencies:** `npm install`
3.  **Environment:** Create a `.env.local` file and add: `GEMINI_API_KEY=your_key_here`
4.  **Run:** `npm run dev`
